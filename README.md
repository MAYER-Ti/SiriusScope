# SiriusScope

Desktop-приложение для потоковой обработки, регистрации и визуализации данных радиотехнической аппаратуры (РТР/РЭР профиль).  
Проект выполняется как дипломная работа на базе Qt 6.9.3 и C++20.

## Цели проекта

- Прием потоковых данных (live) и воспроизведение из файла (replay).
- Первичная обработка и фильтрация событий (режекция, пороги, агрегация).
- Визуализация:
  - **Waterfall** (частота × время, цвет = амплитуда).
  - **Polar/Radar** (индикатор текущего угла антенны и пеленга цели/источника).
- Непрерывная регистрация данных в хранилище без “сеансовости”.
- Метрики и контроль качества потока (очереди, drop-rate, FPS).

## Основные функции (MVP)

- [ ] Эмулятор источника данных (генератор/файл).
- [ ] Live-прием (UDP/TCP).
- [ ] Кольцевой буфер и диспетчер событий.
- [ ] Waterfall-виджет на GPU.
- [ ] Запись в файл + воспроизведение (replay).
- [ ] Базовые фильтры (по амплитуде/частоте/времени).

## Технологический стек

- **Qt:** 6.9.3
- **C++:** C++20
- **UI:** Qt Quick (QML) + кастомные GPU items для тяжёлой графики
- **Build system:** CMake
- **QA:** clang-format, clang-tidy, sanitizers (ASan/UBSan), unit-tests (Qt Test/Catch2)

## Архитектура (верхний уровень)

Поток данных организован как конвейер:

1. **DataSource** (Live/Replayer)  
2. **Ingest Queue / RingBuffer**  
3. **Processing Pipeline** (фильтры/агрегация/детекторы)  
4. **UI Model** (подготовка данных к рендерингу)  
5. **Renderers** (Waterfall, Polar)  
6. **Storage Writer** (непрерывная запись)

UI работает в главном потоке. Прием данных, обработка и запись — в отдельных потоках/пуле.
