# Проект: Разработка единого программного комплекса изделия «Сириус»

## 1. Общие сведения

Проект выполняется в рамках научно-исследовательской работы
по направлению 09.03.04 «Программная инженерия».

Объект исследования:
Изделие «Сириус» — комплекс радиотехнической разведки (РТР).

Назначение комплекса:
- обнаружение и приём сигналов радиотехнических средств;
- измерение частотных, временных и амплитудных параметров;
- пеленгование источников радиоизлучений;
- распознавание сигналов;
- регистрация и передача информации в сопряжённые системы.

## 2. Цель разработки

Разработка единого программного комплекса, объединяющего:

- функции управления изделием;
- функции обработки и анализа сигналов;
- визуализацию спектрально-временных характеристик;
- отображение пеленга;
- подсистему регистрации и хранения данных.

Цель — устранение разрозненности существующих программ
«ПО Управление» и «ПО Анализ» и формирование
единой архитектурно целостной системы.

## 3. Ключевые требования предметной области

### 3.1 Реальное время
- Обновление визуализации без блокировки UI.
- Непрерывный приём данных.
- Минимизация задержек обработки.

### 3.2 Большие объёмы данных
- Поддержка расширенного частотного диапазона.
- Буферизация потоков.
- Возможность долговременной регистрации.

### 3.3 Визуализация
- Центральный элемент — спектрально-временное представление (водопад).
- Панорамный спектр.
- Таблица параметров сигналов.
- Круговой индикатор пеленга.

### 3.4 Архитектурная целостность
- Единая модель данных.
- Разделение вычислений и интерфейса.
- Модульность.
- Масштабируемость.

## 4. Технологический стек

Ядро обработки:
- C++
- многопоточность
- эффективные структуры данных

Интерфейс:
- Qt 6
- QML
- GPU-ускоренная визуализация

Система хранения:
- файловая запись потоков
- журналы событий
- возможность расширения до БД

## 5. Архитектурные принципы

1. Разделение ответственности (Separation of Concerns).
2. Изоляция DSP от UI.
3. Асинхронная передача данных.
4. Единая модель сигналов.
5. Возможность тестирования модулей независимо.

## 6. Основные компоненты

DSP Layer:
- SignalReceiver
- SignalProcessor
- DirectionFinder

Core:
- DataModel
- EventDispatcher
- RecordingManager

UI:
- WaterfallView
- SpectrumView
- AntennaIndicator
- TargetTracker
- SignalTableView

## 7. Ограничения

- Запрещено размещать вычислительную логику в QML.
- Запрещено блокировать GUI поток.
- Запрещено смешивать слои архитектуры.
- Все вычисления должны выполняться вне UI-потока.
